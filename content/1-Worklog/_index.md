**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Team onboarding, AWS fundamentals, and project topic discussion](1.1-week1/)

**Week 2:** [Finalize project direction and map AWS services for implementation](1.2-week2/)

**Week 3:** [Database design, normalization, and scalable ERD baseline](1.3-week3/)

**Week 4:** [Modular project structure and Prisma ORM integration](1.4-week4/)

**Week 5:** [EC2 deployment demo and promotion/admin API development](1.5-week5/)

**Week 6:** [RDS setup, Prisma cloud connection, and admin feature stabilization](1.6-week6/)

**Week 7:** [Authentication hardening, authorization improvements, and API reliability](1.7-week7/)

**Week 8:** [Checkout/order reliability improvements and chatbot integration preparation](1.8-week8/)

**Week 9:** [Chatbot implementation kickoff and backend conversation flow](1.9-week9/)

**Week 10:** [Amazon Bedrock integration and function-calling pipeline](1.10-week10/)

**Week 11:** [RAG implementation, chatbot UI, and Amplify deployment setup](1.11-week11/)

**Week 12:** [End-to-end testing, regression testing, and final stabilization](1.12-week12/)
